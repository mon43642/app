# RepairDeck — แอป Windows สำหรับซ่อม/ตรวจสอบมือถือผ่าน ADB

แอปนี้เรียกใช้ `adb.exe` ตัวจริงของ Google ผ่านแอป Electron จึง **ไม่ต้องเล่น Zadig** เหมือนเวอร์ชันเว็บ WebUSB ก่อนหน้านี้

## วิธีที่ง่ายที่สุด — ให้ GitHub คอมไพล์ .exe ให้ (ไม่ต้องลง Node.js, ไม่ต้องพิมพ์คำสั่งเลย)

ถ้าเครื่องคุณ `npm install` ไม่ผ่าน (ติดปัญหาเน็ต/แอนตี้ไวรัสบล็อกการโหลด Electron) ใช้วิธีนี้แทนได้ จะได้ไฟล์ .exe ตัวจริงมาดับเบิลคลิกเปิดได้เลย โดยให้เซิร์ฟเวอร์ของ GitHub คอมไพล์ให้แทน:

1. สร้างบัญชีฟรีที่ https://github.com (ถ้ายังไม่มี)
2. กด **New repository** ตั้งชื่ออะไรก็ได้ เช่น `repairdeck-app` จะเลือก Private ก็ได้ ไม่บังคับ public แล้วกด Create repository
3. ในหน้า repo ที่สร้างใหม่ กดลิงก์ **"uploading an existing file"** (หรือเมนู Add file → Upload files)
4. ลากไฟล์/โฟลเดอร์ **ทั้งหมด** จากซิปนี้ไปวาง (ต้องมีครบ: `main.js`, `preload.js`, `package.json`, `.gitignore`, โฟลเดอร์ `renderer/` ทั้งโฟลเดอร์, โฟลเดอร์ `build/` ทั้งโฟลเดอร์, และ **โฟลเดอร์ `.github/` ทั้งโฟลเดอร์ด้วย** — อันนี้สำคัญที่สุดเพราะเป็นตัวสั่งให้ GitHub คอมไพล์อัตโนมัติ) จากนั้นกด **Commit changes**
5. ไปแท็บ **Actions** ด้านบนของหน้า repo จะเห็นงานกำลังรันอยู่ (จุดสีเหลืองหมุนๆ) รอประมาณ 3–5 นาทีจนกลายเป็นเครื่องหมายถูกสีเขียว
6. คลิกเข้าไปในรันที่เสร็จแล้ว เลื่อนลงไปด้านล่างสุดของหน้า จะเห็นหัวข้อ **Artifacts** กดดาวน์โหลด `RepairDeck-windows-exe` จะได้ไฟล์ zip มา แตกออกมาจะเจอไฟล์ `.exe` ตัวจริง ดับเบิลคลิกเปิดได้เลย **ไม่ต้องลงอะไรเพิ่มเลยแม้แต่ adb.exe** เพราะ workflow จะโหลด Android Platform Tools มาฝังในตัว exe ให้อัตโนมัติ

ไฟล์ .exe ที่ได้จากวิธีนี้คือไฟล์เดียวกับที่ build เองจะได้ ไม่ต้องติดตั้งอะไร (เป็นแบบ portable) เสียบมือถือแล้วเปิดแอปใช้งานได้ทันที

ทุกครั้งที่แก้โค้ดแล้วอัปโหลดทับใหม่ (Upload files อีกครั้ง + Commit) GitHub จะคอมไพล์ใหม่ให้อัตโนมัติทุกครั้ง

## วิธี build เองในเครื่อง (ถ้าอยากลองหรือ npm install ผ่านปกติ)

## สิ่งที่ต้องมีก่อนเริ่ม

1. **Node.js** (เวอร์ชัน LTS) — ดาวน์โหลดจาก https://nodejs.org แล้วติดตั้งตามค่า default ทั้งหมด ติดตั้งเสร็จให้ปิด-เปิด Command Prompt ใหม่
2. **Android Platform Tools (adb.exe)** — ดาวน์โหลดจาก https://developer.android.com/tools/releases/platform-tools (เลือกไฟล์สำหรับ Windows) แตกไฟล์ zip ออกมาเก็บไว้ที่ไหนก็ได้ จะได้โฟลเดอร์ชื่อ `platform-tools` ที่มี `adb.exe` อยู่ข้างใน

## ขั้นตอนรันครั้งแรก (โหมดทดสอบ ยังไม่ทำเป็น .exe)

เปิด Command Prompt แล้ว `cd` ไปที่โฟลเดอร์นี้ (โฟลเดอร์ที่มีไฟล์ `package.json`) จากนั้นรันทีละคำสั่ง:

```
npm install
npm start
```

แอปจะเปิดเป็นหน้าต่างขึ้นมา ไปที่แท็บ **"ตั้งค่า"** ด้านซ้าย ถ้าแอปบอกว่าหา adb.exe ไม่เจอ ให้กด "เลือกไฟล์…" แล้วชี้ไปที่ `adb.exe` ในโฟลเดอร์ platform-tools ที่ดาวน์โหลดไว้ในขั้นตอนที่ 2 แล้วกด "บันทึกและตรวจสอบใหม่"

เสียบมือถือ เปิด USB debugging และปลดล็อกหน้าจอทิ้งไว้ — แอปจะตรวจหาเครื่องเองอัตโนมัติ ครั้งแรกจะมีป็อปอัปขออนุญาตที่หน้าจอมือถือ ให้กด "อนุญาต" (เลือก "จดจำคอมพิวเตอร์นี้" ด้วยจะสะดวกครั้งต่อไป)

## แพ็กเป็นไฟล์ .exe จริง

เมื่อทดสอบแล้วใช้งานได้ดี รันคำสั่ง:

```
npm run build
```

ไฟล์ `RepairDeck.exe` (แบบ portable เปิดได้เลยไม่ต้องติดตั้ง) จะอยู่ในโฟลเดอร์ `dist/`

### อยากให้ .exe มี adb.exe ติดไปด้วยในตัว ไม่ต้องให้คนอื่นโหลดเอง

ก่อนรัน `npm run build` ให้คัดลอกทั้งโฟลเดอร์ `platform-tools` (ที่แตกจาก Google ทั้งโฟลเดอร์ ไม่ใช่แค่ตัว adb.exe ตัวเดียว เพราะมันต้องใช้ไฟล์ .dll ข้างในด้วย) มาวางไว้ในโฟลเดอร์โปรเจกต์นี้ ให้อยู่ระดับเดียวกับ `package.json` แอปจะมองหาไฟล์ใน `platform-tools/adb.exe` ก่อนเป็นอันดับแรกโดยอัตโนมัติเสมอ ไม่ต้องตั้งค่าอะไรเพิ่ม

## โครงสร้างไฟล์

```
main.js          ← โปรเซสหลักของ Electron เรียก adb.exe จริงผ่าน child_process
preload.js        ← สะพานเชื่อม IPC แบบปลอดภัยให้หน้าเว็บเรียกใช้ adb ได้
renderer/         ← หน้า UI ทั้งหมด (HTML/CSS/JS ธรรมดา)
build/icon.ico    ← ไอคอนแอป
package.json      ← ตั้งค่าโปรเจกต์ + ตั้งค่าการแพ็กเป็น .exe (electron-builder)
```

## ถ้า build แล้ว error

ส่วนใหญ่เป็นเพราะดาวน์โหลด Electron binary ไม่สำเร็จระหว่าง `npm install` (เน็ตหลุด/ไฟร์วอลล์บล็อก) ลองรัน `npm install` อีกครั้ง หรือเช็คว่าเน็ตที่ออฟฟิศ/บ้านบล็อก GitHub ไหม
